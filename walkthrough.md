# Codebase Directory Structure

A brief overview of the project's organization and the purpose of each directory.

### Root Directory
- `Online_Positioning_of_a_Drone-Mounted_Base_Station_in_Emergency_Scenarios.pdf`: Primary research paper implemented.
- `README.md`: General project overview, installation, and usage guide.
- `requirements.txt`: Python dependencies for the simulation.
- `instructions.md`: Task-specific notes and instructions.

---

### `docs/`
*Documentation and presentation materials.*
- `codebase_documentation.md`: Technical deep-dive into code logic.
- `ppt/` & `report/`: Presentation slides and formal reports.

---

### `simulation/`
*Core logic and models for the wireless communication simulation.*
- `algorithm.py`: Implementation of drone positioning algorithms.
- `network.py`: Modeling of network structure and node interactions.
- `propagation.py`: Radio wave propagation and path loss models.
- `traffic.py`: Traffic generation and user demand modeling.
- `drone.py` & `antenna.py`: Drone hardware and antenna pattern models.
- `resource.py`: Network resource management (bandwidth, power).
- `config.py`: Global simulation parameters and settings.

---

### `runners/`
*Scripts to execute the simulation in various modes.*
- `simulation_core.py`: Main simulation execution engine.
- `simulation_batch.py`: Script for batch simulation runs.
- `experiments_launcher.py`: Entry point for research experiments.
- `experiment_*.py`: Specific experiment definitions (convergence, energy, etc.).

---

### `tools/`
*Utility scripts for data analysis and visualization.*
- `plotter.py`: Base functions for chart generation.
- `plot_*.py`: Visualization scripts for convergence, energy, and heatmaps.
- `calibrate.py`: Tools for parameter tuning against models.
- `tune_parameters.py`: Optimization scripts for algorithm constants.

---

### `results/`
*Output files generated by the simulation.*
- `plots/`: Directory for generated visualizations.
- `*.csv`: Raw data exports from simulation runs.

---

### `docs/`
*Documentation and presentation materials.*
- `codebase_documentation.md`: Technical deep-dive into code logic.
- `ppt/` & `report/`: Presentation slides and formal reports.

---

### `tests/`
*Verification suite to ensure code correctness.*
- Unit tests for simulation components and models.

---

### `archive/` & `references/`
- `archive/`: Deprecated code and experimental scripts.
- `references/`: Supporting research papers and external docs.
